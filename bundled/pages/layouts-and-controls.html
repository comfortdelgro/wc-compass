What are layouts, what are controls?

Controls

All of smallest components on screen that can be standalone, they will receive input and emit the output, such as: button, slider, progressbar, loading indicator, inputs, tabs...

Layouts

When we have all of the controls but where to put them on? How our controls display on screen follow the UI design or how can we arrange them to complete the screen?
To define where to place the controls, we will need to have common layout components as well. However, most of client teams when they install and reuse our package, they will have to build their own new layouts system to follow the design system. Which will makes inconsistency between our projects and hard to maintain.

So that we need layouts components in our package. There are two different types of layout. One is for portal page, the other one is for business page / website. However, any layouts can contain any other layouts
Layouts are sidebar, header, section, container, group of controls like a input group.

Layouts need to be matched with our design system. Any controls inside them will automatically follow the sizing, spacing between them. For example: When we put image cover into card, it will automatically located on top with rounded corners.
